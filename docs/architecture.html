Request Data Flow

User changes a filter in FilterBar.vue Writes to a shared ref in useFilters. Because the ref is a module-scope singleton, every mounted view sees the change.
View calls getCurrentFilters() Translates UI vocabulary to API vocabulary and drops any value equal to 'all'.
api.js builds a query string and issues an axios GET e.g. GET /api/orders?warehouse=Tokyo&status=Delivered&month=Q1-2025
FastAPI binds query params as Optional[str] The literal string 'all' also means "no filter" server-side, so the convention is enforced on both ends.
In-memory list comprehension filtering apply_filters() then filter_by_month(). Category matching is case-insensitive; the originals are never mutated.
Pydantic validates the response against response_model A JSON data file whose shape drifts from its model produces a 500 here — not a silent pass.
Vue assigns the payload to a ref; derived values are computed Money renders through formatCurrency(amount, currentCurrency) — amounts are stored USD and converted at display time.

Tech Stack

Frontend

  • Vue 3.4 — Composition API, <script setup>, scoped styles per component
  • vue-router 4 — six routes declared inline in main.js, HTML5 history mode
  • Vite 5 — dev server on :3000, build output client/dist/
  • axios 1.6 — only HTTP client
  • No state library — composable singletons stand in for Pinia/Vuex
  • No chart library — every chart is hand-written SVG; layout is CSS Grid

Backend

  • FastAPI ≥0.110 — single module, ~310 lines, no routers or services split
  • Pydantic 2 — request and response validation
  • uvicorn — served on 0.0.0.0:8001
  • uv — dependency and virtualenv management; Python ≥3.11
  • No ORM, no database, no cache, no auth

Testing

  • pytest + FastAPI TestClient, backend only — no frontend test runner
  • Three suites: test_inventory.py, test_dashboard.py, test_misc_endpoints.py
  • conftest.py puts server/ on sys.path and exposes a client fixture — no running server needed
  • tests/ has no pyproject.toml, so --project ../server is required

Internationalization & currency

  • Hand-rolled useI18n — dot-path t('a.b.c') lookup with English fallback
  • Locales en / ja in locales/*.js; persisted to localStorage['app-locale']
  • Currency is derived from locale, never chosen separately: en → USD, ja → JPY
  • All amounts stored as USD; converted at a hardcoded rate of 150

Design Constraints & Known Sharp Edges

Structural

  • State is not persisted anywhere — a server restart reloads every fixture from disk.
  • All filtering is Python list comprehension over full lists; fine at this scale, linear in record count.
  • Changing a JSON file's shape without updating its Pydantic model 500s the endpoint at response validation.
  • views/Backlog.vue is dead code — no route references it. Backlog data surfaces through Dashboard.vue.

Correctness

  • v-for keys must be stable IDs (sku, id, month) — never the array index.
  • Mock data contains rows with missing or invalid dates; validate before calling .getMonth().
  • Passing month to /api/inventory is a silent no-op, not an error.
  • Revenue goals used by Dashboard and Reports: $800K monthly, $9.6M YTD.

Security posture — demo only

  • No authentication or authorization; useAuth returns a mock user.
  • CORS is allow_origins=["*"] with credentials enabled.
  • No rate limiting, no input size limits, no HTTPS.
  • Repository is public — no credentials, internal hostnames, or private registry URLs may be committed.

Conventions to preserve

  • New filterable endpoints reuse apply_filters() / filter_by_month() rather than re-implementing.
  • Never build filter objects by hand — call getCurrentFilters().
  • Never render a hardcoded $ — go through formatCurrency().
  • Palette is slate/gray (#0f172a, #64748b, #e2e8f0); status colors green/blue/yellow/red. No emojis in UI.
